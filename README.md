## 📘 Selected Projects

> My engineering process begins with **observation**. I identify
> "manual chaos" within organizations and foundations, then architect bespoke technical solutions to solve practical needs and ensure long-term scalability.

### Wix Content Intelligence Pipeline

- **The Scenario:** Fragmented organizational data (events, blogs, CMS) that is difficult to analyze.
- **The Solution:** A robust data ingestion pipeline that extracts real-world data via API and executes automated analysis to provide actionable insights.
- Explore the repo: <a href="https://github.com/beatanemeth/wix-content-intelligence-pipeline" target="_blank">Wix Content Intelligence Pipeline</a>

---

### InsightHubAI — The Archive Assistant

- **The Scenario:** Institutional memory is often lost in old meeting notes and articles.
- **The Solution:** An "Archive Assistant" (Digital Memoir) that preserves every event, meeting, and writing. It allows non-technical staff to generate complex weekly and yearly reports through a
  predefined set of AI tools (MCP), bypassing the need for manual data processing.
- Explore the repo: <a href="https://github.com/beatanemeth/ai-engeneering-study-mcp" target="_blank">Making AI Queries Work: From RAG’s Limits to MCP in Practice</a>

---

### ContentNavigatorAI — The Knowledge-Based Browser

- **The Scenario:** Teams struggle to find specific information within a vast, subject-heavy CMS.
- **The Solution:** A specialized Knowledge Assistant that allows foundations to "browse" their own internal knowledge base via a unified chat interface. It acts as a proficient subject-matter expert, turning static content into an interactive browser app.
- Explore the repo: <a href="https://github.com/beatanemeth/ai-engineering-custom-wix-data-chat" target="_blank">LangChain RAG Project: Unified Custom Data Chat</a>

---

### From Manual Chaos to Modular Automation

- **The Scenario:** Fragile, manual workflows for event management and member communication.
- **The Solution:** An event-driven automation architecture bridging Wix with external ecosystems (Google Sheets, Stripe). I replaced manual tracking with robust, error-tolerant n8n workflows that handle the organizational "operating system" automatically.
- Explore the repo: <a href="https://github.com/beatanemeth/n8n_wix_events" target="_blank">
  From Manual Chaos to Modular Automation: Building a Scalable n8n Workflow for a Foundation</a>
